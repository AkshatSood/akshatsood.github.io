Image Super Resolution using GANs and CNNs

Python (OpenCV, Matplotlib, NumPy, Pandas, TensorFlow, scikit-learn, Seaborn)


Single-image super-resolution (SR) is an important computer vision challenge that aims to reconstruct high-resolution images from low-resolution ones. Various statistical algorithms were employed for this purpose until deep learning models could be used to address this in a much more efficient manner. This report1 details the research and deep learning models that were used to evaluate their SR performance against benchmark datasets, using common image quality assessment (IQA) metrics such as peak signal-to-noise ratio (PSNR), structural similarity index (SSIM), and Blind/Referenceless Image Spatial Quality Evaluator (BRISQUE) score.

The goal of single-image super-resolution (SR) is to reconstruct a high-resolution (HR) image from a low-resolution (LR) image. It is a classical inverse problem in computer vision that is undetermined in nature, with no single unique solution. This process is incredibly useful in situations where HR images are required for analysis, but only LR or low-quality images are available; such as medical diagnosis, satellite images, surveillance, astronomical images, and video analysis and enhancement. Super-resolution has widespread application in a multitude of industries. The increase in research and development in this field in the last few years has made this an extremely challenging and complex task efficient and improved its quality.

There are various techniques that can be used for image super-resolution. These range from classical interpolation-based statistical algorithms to deep learning approaches which employ models such as convolutional neural networks (CNNs) and generative adversarial networks (GANs). This assignment aims to assess some of these single-image SR models using popular benchmark datasets for training and evaluation.

This report summarises the processes followed to evaluate the performance of the single-image super-resolution deep learning models like SRCNN, FSRCNN, and ESRGAN models against benchmark datasets and classical algorithms, on various scaling factors. Although one of the models trained in this project achieves results comparable to industry standards, computing hardware limitations restricted the performance of the trained models. Thus, in order to improve the quality of models trained in this assignment, a much larger number of generated patches (with rotations) need to be used. Despite the adaptability and versatility of state-of-the-art models used in this assignment, there are still scenarios in which the perceptual quality of upscaled images is lacking. Furthermore, most industry standard models are trained on images downscaled via bicubic interpolation. The applicability of extending this to other method needs to be studied for its potential to increase the models’ adaptability to distortions and noise.

Program Analysis and Robustness to Machine Learning Performance Time Decays

Python, Azure

Android has become one of the leading mobile operating system platforms in the world. As of September 2020, there are over 3 million applications available on the Google Play Store. Considering that applications on Android are available via other sources as well (for instance, torrents, one-click hosters, websites, blogs or alternative app markets), the total number of applications available for the Android platform is much higher. In the android landscape, malware represents a significant proportion of all applications. Various sources estimate between 6-10% of android applications are malicious and potentially harmful

Due to constant development of new strains and variants of malware, the composition of malware found in realistic settings is invariably evolving. These changes can lead to inconsistencies in the performance of malware classifiers. It is infeasible to repeatedly retrain classifiers on newer strains of malware. This is due to the high cost of identifying and labelling the malware as well as the computational cost of retraining and distributing the updated classifier. Thus, there is a need to model malware classifiers which can stay effective against the evolving malware landscape. This can be achieved by identifying and studying program analysis methodologies in order to create robust abstractions of the applications.

Android is one of the world's leading tablet and smartphone operating systems. As of 2019, more than 1.6 billion devices worldwide support Android. which accounts for over 70\% of the mobile operating system share worldwide. Research shows that the number of total malware by 2020 is estimated at over 1.1 billion, which has increased 18 times over the last 10 years. Moreover, there have been an average of more than 10 million new malware per month in 2020.

As new forms of malware are being generated, it is important to assess whether models which have been trained on older data sets, can effectively and accurately classify newer applications as malicious or benign. An issue in the Android malware classification realm, is that malware classifiers are not evaluated in settings which represent real world deployments. In order to successfully assess the effectiveness of a classifier in a realistic setting, it is important to evaluate the robustness to temporal decay.

In addition to temporal considerations during classification, the ratio of malware to goodware (or non-malicious) in realistic scenarios should be enforced during evaluation. This ratio is domain specific. Reports suggest that most Android applications are non-malicious.

Thus, there is an urgent need for robust, efficient and spatial-temporal aware malware classification solutions.

The project is research oriented and focuses on evaluating various program analysis methodologies in the malware classification realm. To this end, static program analysis algorithms are considered for this project. The developmental scope of the project will cover the execution of control flow analysis of Android applications, generations of graph vector embeddings and modelling of a malware classifier using these abstractions. Furthermore, the project will evaluate the effectiveness of different abstraction methodologies for classification. With a focus on robustness of classification to changes in malware landscape over time, the project will also evaluate the classifier with time-aware data splits.

There are numerous static program analysis methodologies. However, this project will exclusively focus on control flow analysis and extensions thereof. The methodologies implemented in this project have been selected after careful consideration and are in no way an exhaustive list of approaches.

Due to the research oriented nature of the project, the development of a sophisticated malware detection system is not within the scope of the project. Even though the modelling and development of a classifier is within the scope of the project, the user interface and experience design for the same are not.

Markov Decision Process Solver for the Pac-Man Environment

This coursework is set in a modern emulation of the 1980 maze chase arcade game of Pac-Man. In which the main agent (Pac-Man), explores the maze with the objective of consuming all the available food, while avoiding (or attacking) hostile (or frightened) ghosts. Near the corners of the maze are capsules which (on consuming) provide Pac-Man with the temporary ability to eat the ghosts, that is, change the states of the ghosts from hostile to frightened. Once Pac-Man consumes a ghost, the ghost returns to the "ghost home", where it is regenerated in its hostile state.

The aim of the project is to implement a MDP-solver to control Pac-Man's movements and win games in two different mazes (layouts). A win is defined as the state in which PacMan has eaten all the food on the map. A loss is defined by the state in which a hostile ghost has touched Pac-Man. For the project, the score of the game of is irrelevant; the aim is to win as many games as possible.

After various testing scenarios, I was able to determine the optimal values for the variables in the code. Using these values, I was able to get a win rate of 0.603 and 0.600 for smallGrid and mediumClassic respectively; thus, placing within the Epic and Legendary Pac-Man Skill Levels (according to the marksheet). However, due to the low number of runs during the marking process and randomness in win rates, the results during marking may differ.

Even though an Markov Decision Process (MDP) agent is able to produce high win rates for the Pac-Man environment, it is not consistent. There is a high degree of randomness and variation in this particular implementation of the algorithm. For consistent win rates (or even high scores), other algorithms and approaches (like ad-hoc ghost avoidance) should be explored.
